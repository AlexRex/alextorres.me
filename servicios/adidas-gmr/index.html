adidas GMR — datos de sensores a escala

El problema

adidas GMR metió un sensor de movimiento dentro de una plantilla de bota de fútbol. Cada sesión produce flujos de datos IMU en crudo que hay que ingerir, validar, procesar y devolver a un juego móvil — de forma fiable, para una base de usuarios global.

En qué trabajé

La plataforma de datos serverless detrás del producto, construida en AWS como decenas de pequeños servicios y lambdas.

  • Pipelines de ingesta y validación de datos de sensores, con esquemas protobuf/avro como contrato entre servicios.
  • Sincronización de los flujos de sensores con vídeo para etiquetado y desarrollo de algoritmos.
  • Servicios de apoyo: gestión de usuarios, notificaciones, gestión de campañas y sesiones, tooling de CI/CD.

El resultado

Un producto de consumo de una marca global, lanzado con EA — y la experiencia de construir pipelines de datos que no pueden perder ni una sola sesión.
