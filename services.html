What I do

I'm an independent software consultant. I build custom software for companies whose product runs on complex data: transport networks, vehicle fleets, sensor streams.

  • Maps and geodata: interactive map libraries, geographic and schematic network maps, self-hosted vector tiles (no Google/Mapbox fees), GTFS tooling.
  • Data platforms and backend architecture: serverless pipelines on AWS, splitting monoliths into services, architecture analysis before touching a line of code.
  • Custom tools: internal ops tooling, browser-based PDF/print generation, editors for domain data.

How I work

  1. Understand first. Before writing code I map the problem. For a migration that meant a full set of architecture documents the client still keeps; for a new map, the network data and how riders use it.
  2. Something working, early. A live demo in the first weeks, not a slide deck. Every case below has one you can click.
  3. Deliverables you can use without me. Documentation, editing modes, print-ready output. My job is done when your team doesn't need me anymore, not when the invoice goes out.

I take fixed-scope projects, ongoing collaborations, and embedded work inside your team, working remotely from Spain.
