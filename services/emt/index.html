EMT — network map and printed infographics

The problem

A public bus operator needs its network to be understandable in two very different places: on the web, where riders explore lines and stops interactively, and on the street, where every stop and pole carries printed infographics that must stay in sync with the actual network.

What I built

Two connected systems sharing the same network data:

  • An embeddable map library — a JavaScript library (single-file bundle, plain JS API) that any of the operator's sites can drop in to render the interactive map of lines and stops.
  • An infographics engine — the browser as a layout engine: a slot-based design system composes the infographic for each stop and pole, and an API renders it to print-ready PDF. When the network changes, the printed material regenerates instead of being redrawn by hand.

The result

The infographics generated by this system are printed and installed at real bus stops, and the same source of truth powers the interactive map — web and street never disagree.
